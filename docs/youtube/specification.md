# YouTube版 9コマ（仮） 要件定義および基本仕様書

## 1. プロジェクトの目的とコアコンセプト
既存の『私を構成する9つのマンガ（9coma）』の基本思想を受け継ぎつつ、**「私を構成する9つのYouTube動画・チャンネル」**を作成・シェアできる並行サービスを立ち上げる。
最大の特長として、ユーザーの自由な表現を支えるため**「動画」と「チャンネル」の両方のURL入力を許容するハイブリッド仕様**とする。

## 2. ディレクトリ構造の分断方針（完全セパレート・アーキテクチャ）
既存のマンガ版（縦長・B6判基準）の精緻なUIやOGP生成ロジックを破壊・汚染しないよう、UIおよびルーティングの根幹を「16:9」専用として完全にセパレートする。

### 構成案
*   **ルーティング（URL空間）**
    *   マンガ版: `src/app/` (既存維持)
    *   YouTube版: `src/app/yt/` (トップページ), `src/app/yt/l/[id]/` (シェアリストページ)
*   **UIコンポーネント**
    *   `src/components/youtube/` を新設し、16:9専用の `YtGrid.tsx` や `YtSlot.tsx`、専用検索モーダル `YtSearchModal.tsx` などを全てこの配下に隔離する。
*   **OGP画像生成（動的画像共有）**
    *   `src/app/api/og/youtube/route.tsx` を新設。16:9のグリッド3x3＝全体の比率も完璧な16:9となるため、Twitterカードに最適化された専用SVG生成ロジックを組む。

## 3. 取得フローと外部API戦略（URL入力主軸）
YouTube Data API v3 の厳しい無料枠（1日1万ユニット / 検索1回につき100ユニット消費）を回避するため、キーワード検索への依存を極力減らし、**「動画URL または チャンネルURL をユーザーに直接貼り付けさせる」**手法を主軸とする。

### ① 動画 URL が入力された場合 (`youtube.com/watch?v=...` 等)
*   **取得方法**: 無料で利用可能な YouTube 公式の `oEmbed API`、または Data API の `videos` エンドポイント（消費1ユニット/回）を利用。
*   **取得データ**: 動画タイトル、チャンネル名、サムネイル画像（16:9の高画質版）。

### ② チャンネル URL が入力された場合 (`youtube.com/@...` 等)
*   **取得方法**: URL先のHTMLをサーバーサイドでスクレイピング（`fetch` + 正規表現等）し、`<meta property="og:image">`（チャンネルアイコン）および `<meta property="og:title">` を取得。または Data API の `channels` エンドポイントを利用。
*   **取得データ**: チャンネル名、アバター画像（1:1 正方形または円）。

## 4. UI/UXにおける「比率混在」の解決策
16:9（動画）と 1:1（チャンネル）という全く異なるアスペクト比のアイテムが同じ 3x3 グリッド内に共存するため、マス（枠）は**強制的に16:9に固定**し、チャンネルURLがセットされた枠は以下の「専用デザインスタイル」を適用する。

*   **16:9 マスの背景**: 取得したチャンネルのアバター画像に「強いブラー加工（ぼかし）」と「ダークレイヤー」を重ねて敷き詰める。
*   **マスのメインビジュアル**: 中央に円形にくり抜いたアバター（1:1）を配置。
*   **テキスト**: アバターの下部に白色でチャンネル名を美しく配置。
これにより、黒帯（レターボックス）の発生を防ぎ、極めてモダンで洗練された統一感を生み出す。

## 5. データベース設計 (Firestore)
マンガのデータと混ざらないよう、データの保存先を完全に分断する。

*   **コレクション**: `yt_lists`
*   **保存データモデル**:
    ```typescript
    interface YouTubeListItem {
      id: string;             // FirestoreのドキュメントID
      theme: string;          // リストのテーマ（例: 「私の青春のゲーム実況」など）
      authorName: string;     // 作成者名
      slots: (YouTubeSlot | null)[];  // 長さ9の配列
      createdAt: number;
    }

    interface YouTubeSlot {
      type: 'video' | 'channel'; // クライアント側でのUI分岐用
      url: string;               // 元のURL
      title: string;             // 動画タイトル or チャンネル名
      channelName?: string;      // 動画の場合のアップロード主
      imageUrl: string;          // サムネイル or アイコン画像のURL
      videoId?: string;          // 動画ID（後々の再取得用）
    }
    ```

3.  **Phase YT-3**: 16:9専用の `YtGrid.tsx` および モーダルUIの開発（混在UIのスタイリング）
4.  **Phase YT-4**: Firetoreへの保存 (`yt_lists`) および リスト個別画面 (`/yt/l/[id]`) の構築
5.  **Phase YT-5**: YouTube版専用 OGP画像ジェネレーター (`/api/og/youtube`) の構築
