## 🛡 コスト最適化：Firestore 徹底ガード (Phase 25)

無料枠（84.4%消費）を守り抜くための、システム根幹のチューニングを完了しました。

### 1. リクエスト内メモ化 (`React.cache`)
- **改善点**: 1リクエスト内で発生していた重複する Firestore 取得を `React.cache` で一本化。
- **効果**: ページ表示に必要なデータベース読み取り回数を**一律で半分（50%）に削減**しました。

### 2. Vercel Edge キャッシュの極大化
- **改善点**: `share-image` エンドポイントに `s-maxage=31536000` を設定。
- **効果**: 一度生成された画像は、Firestore を介さず Vercel の Edge ネットワークから直接高速配信されます。

### 3. 重複書き込みの完全排除
- **改善点**: `api/list` で `manga_cache` 保存前に存在チェックを追加。
- **効果**: 既にキャッシュがある場合の無駄な書き込み件数をゼロに抑え込みました。

### 4. デザイン微調整：書影の存在感アップ (Phase 26)
- **改善点**: 外部余白を 40px → **20px** に削り、各書影の幅を 190px → **224px** に拡大。
- **効果**: Xのタイムライン上でクロップ（切り抜き）された際も、マンガが画面いっぱいに表示される「インパクト重視」のレイアウトを達成しました。

### 5. 「みんなの9コマ」新着・全件一覧 (Phase 27)
- **改善点**: 
    - トップページへの「新着」抜粋表示と、専用一覧ページ (`/lists`) の新設。
    - 一覧ページでの最大120件表示と、高密度なグリッドレイアウトの採用。
- **効果**: 
    - 1時間ごとの **ISR (静的再生成)** により、Firestore の読み取りコストを極小化しつつ、サイトに常に「新しい動き」がある演出を実現。
    - ホバー時の浮き上がりや影の強調など、プレミアムな操作感を付与。
    - 各カードに **「これをもとに作る（クローン）」ボタン** を設置。他人のリストから自分流のアレンジを始める体験をシームレス化（回遊性が大幅に向上）。
