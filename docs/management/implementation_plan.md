# コマ割り（レイアウト）選択機能の実装 (Phase 8)

「9coma」の基本である9枚のマンガを維持しつつ、特定の作品を大きく表示するなど、ユーザーが表現したいニュアンスに合わせて配置（レイアウト）を選択できる機能を追加します。

## 変更内容

### 1. [Common] 型定義の拡張
- **ファイル**: `src/types/index.ts`
- **内容**: `ComicList` インターフェースに `layoutType` フィールドを追加します。
  - `layoutType`: `'standard'` (3x3), `'featured'` (1大+8小) など。

### 2. [Backend] 保存・取得ロジックの対応
- **ファイル**: `src/app/api/list/route.ts`
- **内容**: POST/GET 時に `layoutType` を Firestore に保存・取得できるようにします。

### 3. [Frontend] レイアウト選択 UI の追加
- **ファイル**: `src/components/HomeClient.tsx`
- **内容**: 
  - ヘッダー付近にレイアウト選択用のボタンまたはアイコンを設置します。
  - 選択したレイアウトに応じて、編集画面のグリッド構成を動的に変化させます（CSS Grid の調整）。

### 4. [Frontend] 閲覧ページでの表示対応
- **ファイル**: `src/app/list/[id]/ListViewClient.tsx`
- **内容**: 
  - 保存された `layoutType` に基づき、グリッドレイアウトを切り替えて表示します。

### 5. [Backend] OGP画像・シェア画像への反映
- **ファイル**: `src/app/list/[id]/share-image/route.tsx`
- **内容**: 
  - Satori (next/og) の Flexbox/Grid レイアウトを `layoutType` によって分岐させ、シェア画像にもレイアウトの個性が反映されるようにします。
