# ホスティング移転に伴う技術的影響分析（公式記録）

Vercel Pro から他のプラットフォーム（特に Cloudflare Pages/Workers）へ移行、あるいは並行運用を検討する際の「できなくなること」「調整が必要なこと」を整理した公式記録です。

## 1. 比較一覧：Vercel vs 他プラットフォーム

| 機能項目 | Vercel (現状) | Cloudflare Pages / AWS 等 | 影響と対策 |
| :--- | :--- | :--- | :--- |
| **OGP 動的生成** | `next/og` (Satori) が完全動作 | 動作するが、フォント設定等に微調整が必要 | OGP 自体は維持可能。ただし Edge 環境のメモリ制限がより厳しくなる。 |
| **ISR (静的再生成)** | 標準機能として完璧に動作 | プラットフォームにより挙動が異なる | Cloudflare では `revalidate` が期待通り動かない場合があり、手動のパージが必要。 |
| **Edge Runtime** | Vercel 独自最適化が強力 | 各社独自の Edge 実装に書き換えが必要 | Vercel 特有の API は使えないため、Web Standard API への準拠が必須。 |
| **QRコード (Node版)** | `nodejs` ランタイムで動作 | Edge 環境では動作不可 | ピュア JS 版への差し替えが必要。 |
| **自動最適化** | 画像最適化 (`next/image`) が標準搭載 | 別途設定または課金が必要な場合が多い | Cloudflare 等では画像変換の別料金が発生する可能性がある。 |

## 2. 移行時に「できなくなること」・「困ること」

### A. OGP 生成の安定性低下
Vercel は `next/og` の開発元であるため、複雑な JSX/CSS を用いた画像生成の互換性が最も高いです。他へ移ると、日本語フォントのレンダリングが崩れたり、生成速度が低下（＝追加の CPU コスト発生）するリスクがあります。

### B. 「1時間ごとの自動更新 (ISR)」の喪失
現在、著者ページなどで設定している `revalidate = 3600` は、Vercel のインフラが裏側で管理しています。Cloudflare 等へ移ると、これが「ただの動的ページ」または「ずっと静的なまま」になり、Firestore への負荷が跳ね上がるか、情報が古くなるかの二択になる恐れがあります。

### C. サーバーレス関数の無料枠管理
Cloudflare Workers 等は非常に安価ですが、リクエスト数や CPU 時間の制限の仕方が Vercel と異なります。現在の「画像生成」のような重い処理をそのまま移すと、今度は Cloudflare 側で「実行時間オーバー」によるエラーが多発する可能性があります。

## 3. 結論としての推奨事項

現状の OGP 機能や ISR（負荷軽減）を維持しつつ、将来の移転コストを下げるためには **「Vercel 特有の機能に依存しすぎず、標準的な軽量コード（ピュア JS ライブラリ等）へ寄せていく」** ことが、アフィリエイト収入が確立するまでの賢い生存戦略となります。
