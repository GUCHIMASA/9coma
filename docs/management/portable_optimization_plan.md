# ポータブル軽量化・最適化計画案

Vercel Pro のリソースを節約しつつ、将来的に Cloudflare Pages 等の他プラットフォームへも最小限の修正で移行できる「汎用性の高い軽量コード」への転換プランです。

## 1. 方針：ポータブル＆スタンダード
- **Web 標準 API への準拠**: Node.js 固有のライブラリ（`fs`, `buffer` 等）への依存を排除し、Fetch API や Crypto 等の Web 標準 API を優先します。
- **ランタイムの共通化 (Edge対応化)**: 制限の厳しい Edge Runtime で動作するコードは、結果として Node.js を含むあらゆるモダンなインフラで高速に動作します。
- **不要な機能の削ぎ落とし**: 処理負荷が高く、ポータビリティを阻害する「サーバーサイドでの QR コード画像生成」を整理または軽量化します。

## 2. 変更内容の内訳

### A. 縦長シェア画像 (`share-image`) の軽量化
- [MODIFY] `src/app/list/[id]/share-image/route.tsx`
  - **QRコードの整理**: 処理が重く Node 依存の強い `qrcode` ライブラリの利用を停止、またはシンプルなテキスト URL 表記に変更し、`Edge Runtime` での動作を可能にします。
  - **ランタイム変更**: `runtime = 'edge'` を指定。

### B. OGP生成ロジックの標準化
- [MODIFY] `src/app/author/[authorName]/opengraph-image.tsx`
- [MODIFY] `src/app/list/[id]/opengraph-image.tsx`
  - **ランタイム共通化**: `runtime = 'edge'` を指定し、Web 標準 API のみに制限することで、実行コストを極限まで下げます。
  - **キャッシュ強化**: `immutable` ヘッダーを付与し、再計算（CPU消費）を長期間抑制します。

## 3. この計画の有用性
- **コスト対効果**: CPU 消費時間を 1/10 以下に削減し、Pro プランの枠を最大限に活かせます。
- **将来の自由度**: Cloudflare 等へ移る際、コードを書き直さずにそのまま動かせる確率が飛躍的に高まります。
- **保守性**: 依存ライブラリが減るため、ビルドエラーやセキュリティリスクが低減します。
