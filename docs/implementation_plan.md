# 9coma - コマ割り（ダイナミックレイアウト）追加実装計画

## 目的
ユーザーが従来の「3x3均等グリッド」に加えて、少年漫画の誌面のような「コマ割りレイアウト (dynamic_01)」を選択し、シェアできるように機能拡張を行います。

## 対象機能・スコープ
1. **データ構造の拡張:** リスト保存時に `layoutType` 情報を保持する。
2. **バックエンド更新:** API (`/api/list`) が `layoutType` を正しく送受信し、過去データには `standard` をフォールバックする。
3. **フロントエンド (トップページ):** レイアウト切り替えUI（トグル）を追加する。
4. **フロントエンド (詳細ページ):**
   - 選択された `layoutType` に応じたCSS Grid構造の出し分け。
   - すべてのコマの画像下部に「マンガのタイトルテキスト（見やすいグラデーション付き）」を表示する。

## 詳細実装ファイル

### 1. `types/index.ts`
- `ComicList` インターフェースに `layoutType?: 'standard' | 'dynamic_01'` を追加します。

### 2. `src/app/api/list/route.ts`
- **POST処理:** リクエストボディから `layoutType` を受け取り、`id` などと一緒にFirestore/モックに保存します。指定がない場合は `'standard'` をデフォルトとします。
- **GET処理:** 読み込み時もデータをそのまま返し、フロントエンド側で `layoutType` に基づき描画します。

### 3. `src/app/page.tsx`
- `layoutType` のState (`'standard' | 'dynamic_01'`) を追加します。
- 「ページを作成してシェア」ボタンの直上に、レイアウトを選択できるトグルUI（あるいはタブUI）を美しいデザインで（CSS変数を活用して）追加します。
- POSTリクエスト時に `layoutType` を送信データに含めます。

### 4. `src/app/list/[id]/ListViewClient.tsx`
- `data.layoutType` を使って描画を分岐します。
- **タイトル表示機能（共通）:** 
  - 各 `<div>` に `position: relative` を設定。
  - 画像の同階層に、下部から黒グラデーション（`linear-gradient`）を敷き、その上に白文字でタイトルを表示するオーバーレイを追加します。長いタイトルは `text-overflow: ellipsis` 等で省略表示にします。
- **Standard (従来通り):**
  - 均等な3x3のグリッド構造を維持しつつ、上記のタイトルテキストを重ねます。
- **Dynamic 01 (見本画像レイアウト):**
  - CSS Grid を使用し、デザイン画像に沿って以下の配置を構築します。
    - 大枠を組んで、左側にSlot 5（特大）、右側にSlot 1, 2（上部並列）、Slot 3, Slot 4（縦積み）を配置。
    - 下段にSlot 9, 8, 7, 6 を並列配置。
    - **斜め線（コマ割り風）の表現:** 雑誌風の斜め線をCSSの `clip-path` を用いて可能な限り見本に忠実に再現します。各コマに `clip-path: polygon(...)` を適用し、重なりや隙間（ギャップ）を微調整して漫画のコマ割りを表現します。
  - 見本画像で 2, 1 や 9, 8, 7, 6 と右から左へナンバリングされている点（漫画の右開きのような表現）も踏まえて、グリッド内の配置順序や `clip-path` の角度を調整します。

## 検証計画 (Verification Plan)

### Automated Tests (自動テスト/ビルド)
- TypeScriptの型エラーがないことを `npm run type-check`（または `npm run build` 時のエラー非発生）で確認する。

### Manual Testing (ローカルでの手動検証)
1. `npm run dev` でローカルサーバーを起動する。
2. トップページ（`http://localhost:3000/`）にアクセスし、9冊のマンガを選択する。
3. 新設されたレイアウト切り替えボタンを操作し、デザインの切り替わりと状態が保持されることを確認する。
4. 「ダイナミック(dynamic_01)」を選択した状態で「ページを作成してシェア」を実行する。
5. 生成されたページ (`/list/[id]`) を開き、以下の表示を確認する：
   - 漫画のコマ割り風CSS配置が見本画像に近似していること。
   - `clip-path` などによって画像が斜めに切り取られ、コマとコマの隙間が漫画風になっていること。
   - 全ての画像の下部に、マンガのタイトルがグラデーション付きで白文字で表示されていること。
6. 「スタンダード(standard)」も同様に作成し、以前と同じ正方形の3x3グリッドで、かつタイトルテキストが表示されていることを確認する。
