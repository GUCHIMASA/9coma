## 🛠️ Step 1: カラーシステムと書影コンテナの基礎実装
**【目的】: 背景色のID管理化と、モダンなコンテナデザインの統一**
1. **カラーセットのID管理 (`src/lib/colors.ts` 等の新設・調整)**:
   - 背景色と最適な文字色（+シャドウ色など）をセットにしたオブジェクトを定義し、**「2桁の通し番号（ID）」**で管理・保存できるようにしてください。
   - 例: `01: { bg: '#FFD600', text: '#1A1A1A' }`、`02: { bg: '#000000', text: '#FFFFFF' }`
   - 対象カラー: デフォルト（黄）、黒、グレー、白、赤、ピンク、青、水色、緑、明るい緑、茶色、ベージュ。
2. **書影（マンガ表紙）コンテナのデザイン統一 (`HomeClient.tsx` 等)**:
   - **枠線の撤廃**: コンテナ外周の罫線（ボーダー）を完全に削除。
   - **シャドウの適用**: 薄く上品なドロップシャドウのみを適用（透明プラ版印刷でも映えるモダンな作り）。
   - **タイトル表示帯の統一**: 書影下部に重なるタイトル表示部分は、**常に「下から上に向かって黒から透明へのグラデーション」**とし、文字色は白に固定。
*※Step 1 完了時点で一度、監督者（Antigravity）に報告を上げ、トップページのUI崩れがないか監査を受けてください。*

## 🖼️ Step 2: 3×3シェア画像の「Xタイムライン（4:3）」極限最適化
**【目的】: スマホシェア時の画像（ListViewClientからの生成画像）をXのタイムラインに完璧に収める**
1. **レイアウトの大前提（既存サイズ維持とXタイムライン重視）**:
   - ベースの画像サイズは不必要に変更せず、Xのタイムラインプレビュー表示に特化する。
2. **主役は書影（マンガ）**:
   - マージン（余白）やオブジェクト間の隙間は極力狭く絞り、画像全体に占める**「書影コンテナの面積」を最大化**してください。
3. **4:3 への完全収納 (`src/app/api/share-image/route.tsx` 等)**:
   - Xのタイムライン等で頻出する「4:3」のトリミングエリア内（画像中央を基準とする横長エリア）に、「9コマのコンテナ全体」と「テーマテキスト」が**1ピクセルの見切れもなく綺麗に収まる**よう調整してください。
   - 具体的には、元の画像サイズの中で、書影コンテナの全体の縦幅、テーマテキストのフォントサイズなどを緻密に計算し、上部のテキストから下部のマンガ群までの重要情報が 4:3 比率内に完璧にパッキングされるよう計算に基づく修正を行うこと。
*※Step 2 完了時点で、生成されたシェア画像のスクリーンショット（または画像URL）を提示し、レイアウト比率の監査を受けてください。*

## 🖼️ Step 3: 動的OGP（1大＋8小テンプレート）レイアウト修正
**【目的】: リスト共有時（`/list/[id]/opengraph-image.tsx`）のレイアウト崩れ修正**
1. **コンテナサイズ比率の修正**:
   - 過去のフォント変更の影響で、右側の8つのコンテナの上下に無駄な余白が発生している問題を修正します。
   - 大枠（左1マス）と小枠（右8マス）の縦幅計算を見直し、右側8マスの上下がピッタリ完全に収まる（無駄な隙間がない）ように `height` と `margin/gap` を再計算・調整してください。
*※Step 3 完了時点で、OGPのプレビュー確認を実施し、全体のタスク完了報告を上げてください。*
