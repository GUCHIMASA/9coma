# 9coma ver.1.0 確定仕様書

## 概要
「9coma」は、ユーザーが自身を形成した「私を構成する9つのマンガ」を検索・選択し、1つのWebページとしてまとめ、SNS（主にX等）でシェアすることができるWebサービスです。

## 本番環境 / インフラ
- **本番環境URL:** https://9coma.com (Hosting URL: https://coma-5555b.web.app)
- **フロントエンド:** Next.js 14 (App Router), React 18, TypeScript
- **ホスティング:** Firebase Hosting
- **データベース:** Firebase Firestore (`lists` コレクション)

## 主要機能と仕様

### 1. マンガ検索機能 (`/api/search`)
- **外部API:** 楽天ブックス総合検索API (v2)
- **検索UI:** 「タイトル」「著者名」「キーワード」の3つの入力欄を備え、入力内容に応じて自動的にデバウンス検索をバックグラウンドで実行する。
- **検索ロジック:** 
  - 楽天APIの仕様に対応するため、バックエンド (`route.ts`) にて入力された3項目を結合し、1つの `keyword` パラメータとしてAPIに送信している。
  - 絶版・品切れの古い作品もユーザーが選択できるよう、楽天APIのリクエストに `outOfStockFlag=1` を付与し、在庫なし作品も取得対象に含めている。
- **アフィリエイト連携:** 検索結果から取得される漫画データには自動的に管理者（開発者）の楽天アフィリエイトリンクが付与され、ユーザーがそれを経由して購入した場合に報酬が発生する仕組み。

### 2. リスト作成機能 (トップページ: `/`)
- **グリッドUI:** 3x3（計9枠）のグリッドデザイン。
- **特等席:** 中央の5番目の枠（Slot 5）は特別な意味合いを持たせるため、背景色が「薄ピンク (`#FFA8B8`) 」で強調表示される。
- **統一テキスト:** プロジェクト全体で「私を構成する9つのマンガ」という表記に統一されている。
- **データ保存:** 任意の作成者名とともに、選んだ最大9冊のマンガ情報と、任意で設定された「テーマ（タグ）」(e.g., `#人生を変えた等`) がFirestoreに保存され、固有のアピールURL (`/list/[id]`) が生成される。
- **自動ドラフト機能:** ユーザーが選択したマンガ、入力した名前、選択したテーマは、ブラウザの `localStorage` (`draft_9coma`) に随時自動保存され、次回アクセス時やシェア画面から戻ってきた際に即座に状態が復元される。
- **リストのクローン仕様:** URLクエリに `?clone=[id]` が付与された状態でアクセスされた場合、対象のリストデータをAPI経由で取得し、初期状態として展開する機能を持つ（他ユーザーのリストを元に作成開始できる機能）。

### 3. リスト閲覧・シェア機能 (`/list/[id]`)
- **SSR (Server-Side Rendering):** X (旧Twitter) などのSNSクローラーが確実にメタデータ (OGP) を読み取れるよう、個別ページはサーバーサイドレンダリングで情報を先行返却している。
- **表示内容:** 選ばれた9つのマンガと作成者名を表示。
  - **書名表示:** UI改善として、画像だけでなく各コマ（書影）の下部に半透過の黒背景でマンガのタイトルテキスト（最大2行で省略）を表示するようになり、利用者の視認性を向上。
  - **テーマバッジ表示:** ユーザーが「テーマ」を設定していた場合、見出し付近にそのテーマに応じた専用のグラデーションカラーバッジ（`#〇〇編`）を表示。
- **拡散・編集導線:**
  - **SNSシェア:** X（Twitter）、URLコピー等のシェア導線を実装。Xへのシェア時、テーマが設定されている場合は『「〇〇」な〇〇を構成する9つのマンガ』のようにテキストが自動変化し、デフォルトで `#9coma #9koma #My9manga` のタグが付与される。
  - **編集再開（自分用）:** 「戻る」ボタンにより、トップページへ遷移し、`localStorage` のドラフト機能により安全に編集を再開できる。
  - **クローン作成（他人用）:** 「✨ コピーして編集」ボタンにより、閲覧者がこのリストの内容をそのまま流用して（`/?clone=[id]`）新たなリスト作成を開始できるバズ導線が実装されている。

### 4. OGP (Open Graph Protocol) 生成機能
- **トップページ用静的OGP:** トップページ自体がシェアされた場合は、用意された専用の静的OGP画像 (`src/app/opengraph-image.png`) およびそのデザイン (`9coma_ogp2.png/psd`準拠) を表示。
- **リストページ動的OGP (`/list/[id]/opengraph-image`):**
  - **画像サイズ:** 1200 x 630
  - **使用技術:** `next/og` (ImageResponse) による完全な動的生成。
  - **デザイン・レイアウト要素:** 
    - 背景は黄色系のグラデーション（`linear-gradient(180deg, #FFD600 0%, #FFB300 100%)`）。
    - **左側領域 (幅352px):** 上端部に「9COMA.COM」のダークバッジを配置し、その下部に**特等席である5番目のマンガ（Slot 5）を特大サイズ**で配置（未指定時は薄ピンク `#FFA8B8`）。
    - **右側領域 (幅768px):** 残りの8冊のマンガを「4列×2行」のグリッドで配置（上段にSlot 1〜4、下段にSlot 6〜9）。各画像の下部に書名が表示される構造。

### 5. デザインシステム
- **テーマカラー:** 「黄色 (`#FFD600` 系統)」をメインカラーに据えた、明るくポップなデザイン。
- **スタイリング:** Vanilla CSS (`globals.css`) を使用し、CSS変数にて全体の色調・丸みを一元管理している（Tailwind等のフレームワークは不使用）。

### 6. トラッキング・SEO・法的ページ設定
- **Google Analytics (GA4):** `src/app/layout.tsx` 内にGoogle公式の最適化されたスクリプト (`next/script`) を実装。環境変数 `NEXT_PUBLIC_GA_ID` を定義するだけで自動でアクセス解析を実行。
- **メタデータ:** Next.js標準の metadata 設定オブジェクトで `metadataBase` を利用し、絶対パスによるURL解決と正規化を行っている。
- **プライバシーポリシー**: アプリケーションを健全に運用するため `/privacy` ページを設置。GA4や楽天アフィリエイトの利用について明記し、全ページ共通のフッターからアクセスできる構造にしている。

## 今後の開発・運用ルール（厳守事項）
ver.1.0リリースおよび実際のユーザー稼働状況を鑑み、今後は**本番環境の破壊を防ぐため以下のワークフローを必ず守ること。**

1. **ローカルプレビューの徹底:** 新機能の追加、UI変更やソースコード修正を行った際は、勝手に Firebase やリモートへの `git push`・`deploy` を実行しない。
2. **ユーザーへの事前確認:** 修正後は必ず開発サーバー(`npm run dev`等)を立ち上げ、ユーザー自身がローカルブラウザで操作・表示の確認ができる状態を案内する。
3. **明示的な許可:** ユーザーからの「確認した、デプロイして良い」という明確な許可を得た後でのみ、デプロイメントのプロセスを実行する。
