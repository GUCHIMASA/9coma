# 9coma ver.1.0 確定仕様書

## 概要
「9coma (ナインコマ)」は、ユーザーが自身を形成した「私を構成する9つのマンガ」を検索・選択し、1つのWebページとしてまとめ、SNS（主にX等）でシェアすることができるWebサービスです。

## 本番環境 / インフラ
- **本番環境URL:** https://9coma.com (Hosting URL: https://coma-5555b.web.app)
- **フロントエンド:** Next.js 14 (App Router), React 18, TypeScript
- **ホスティング:** Firebase Hosting
- **データベース:** Firebase Firestore (`lists` コレクション)

## 主要機能と仕様

### 1. マンガ検索機能 (`/api/search`)
- **外部API:** 楽天ブックス総合検索API (v2)
- **検索項目:** 「タイトル」「著者」「その他キーワード」を用いて柔軟に検索が可能。
- **アフィリエイト連携:** 検索結果から取得される漫画データには自動的に管理者（開発者）の楽天アフィリエイトリンクが付与され、ユーザーがそれを経由して購入した場合に報酬が発生する仕組みとなっている。

### 2. リスト作成機能 (トップページ: `/`)
- **グリッドUI:** 3x3（計9枠）のグリッドデザイン。
- **特等席:** 中央の5番目の枠は特別な意味合いを持たせるため、背景色が「薄ピンク (`#FFA8B8`) 」で強調表示される。
- **データ保存:** 任意の作成者名とともに、選んだ最大9冊のマンガ情報がFirestoreに保存され、専用のURL (`/list/[id]`) が生成される。

### 3. リスト閲覧・シェア機能 (`/list/[id]`)
- **表示内容:** 追加された9つのマンガと作成者名を表示。
- **シェアボタンリンク:** X（Twitter）へのシェアリンクには、デフォルトで以下のハッシュタグが付与される。
  - `#9coma #9koma #私を構成する9つのマンガ`

### 4. OGP (Open Graph Protocol) 生成機能
- **トップページ用OGP:** トップページ自体がシェアされた場合は、専用の静的OGP画像 (`src/app/opengraph-image.png`) を表示。
- **リストページ動的OGP (`/list/[id]/opengraph-image`):**
  - **画像サイズ:** 1200 x 630
  - **使用技術:** `next/og` (ImageResponse) による動的生成。
  - **レイアウト図:**
    - **左側領域:** 上端に「9COMA.COM」のバッジ、その下に大きく5番目のマンガ（背景色 `#FFA8B8`）を配置。
    - **右側領域:** 残りの8冊（1, 2, 3, 4 を上段、6, 7, 8, 9 を下段）を4列2行で配置。それぞれ画像の下には書名が表示される（長い場合は省略）。

## 今後の開発・運用ルール（厳守事項）
ver.1.0リリースおよび実際のユーザー稼働状況を鑑み、今後は**本番環境の破壊を防ぐため以下のワークフローを必ず守ること。**

1. **ローカルプレビューの徹底:** 新機能の追加、UI変更やソースコード修正を行った際は、勝手に Firebase やリモートへの `git push`・`deploy` を実行しない。
2. **ユーザーへの事前確認:** 修正後は必ず開発サーバー(`npm run dev`等)を立ち上げ、ユーザー自身がローカルブラウザで操作・表示の確認ができる状態を案内する。
3. **明示的な許可:** ユーザーからの「確認した、デプロイして良い」という明確な許可を得た後でのみ、デプロイメントのプロセスを実行する。
